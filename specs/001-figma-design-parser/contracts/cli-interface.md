# CLI Interface Contract: figma-scaler

**Branch**: `001-figma-design-parser` | **Date**: 2026-02-27

## Команда: `parse`

Основная и единственная команда v1. Парсит Figma-файл и генерирует
дизайн-токены.

### Синтаксис

```
figma-scaler parse <fileId> [options]
```

### Аргументы

| Аргумент | Обязательный | Описание |
|----------|-------------|----------|
| `fileId` | Да | Figma file ID или полный URL файла |

### Опции

| Опция | Короткая | Обязательная | Default | Описание |
|-------|----------|-------------|---------|----------|
| `--token <token>` | `-t` | Да* | `$FIGMA_TOKEN` | Figma Personal Access Token |
| `--output <dir>` | `-o` | Нет | `./figma-tokens` | Директория для выходных файлов |
| `--page <name>` | `-p` | Нет | — | Парсить только указанную страницу |
| `--node <id>` | `-n` | Нет | — | Парсить только указанную ноду и её потомков |
| `--include-hidden` | — | Нет | `false` | Включать скрытые слои |
| `--format <fmt>` | `-f` | Нет | `all` | Формат вывода: `all`, `json`, `css`, `context` |

*Если переменная `FIGMA_TOKEN` установлена, опция `--token` не обязательна.

### Примеры

```bash
# Базовый парсинг
figma-scaler parse ABC123XYZ --token figd_xxx

# С фильтрацией по странице
figma-scaler parse ABC123XYZ -t figd_xxx --page "Design System"

# Конкретная нода, вывод только JSON
figma-scaler parse ABC123XYZ -t figd_xxx --node "1:23" --format json

# Из URL файла, токен из env
export FIGMA_TOKEN=figd_xxx
figma-scaler parse "https://figma.com/file/ABC123XYZ/My-File"

# С включением скрытых слоёв, кастомная директория
figma-scaler parse ABC123XYZ -t figd_xxx -o ./tokens --include-hidden
```

## Выходные файлы

При `--format all` (по умолчанию) генерируются:

```
<output-dir>/
├── colors.json          # Цветовые токены (DTCG формат)
├── typography.json      # Типографические токены
├── spacing.json         # Spacing-токены
├── border-radius.json   # Border-radius токены
├── shadows.json         # Тени
├── gradients.json       # Градиенты
├── components.json      # Компоненты с layout и вариантами
├── design-system.css    # CSS Custom Properties
├── CONTEXT.md           # AI-контекст файл
├── manifest.json        # Метаданные генерации
└── images/              # Скачанные изображения (создаётся при наличии IMAGE fills)
    └── {node_id}.png    # Формат: "1-23.png" (`:` → `-`)
```

## Exit Codes

| Код | Значение |
|-----|----------|
| 0 | Успех |
| 1 | Ошибка аргументов (невалидный fileId, отсутствует token) |
| 2 | Ошибка API (невалидный token, файл не найден, rate limit) |
| 3 | Ошибка записи файлов (нет прав, диск полный) |

## JSON формат вывода (DTCG)

Пример `colors.json`:

```json
{
  "primary-500": {
    "$type": "color",
    "$value": "#2563eb",
    "$extensions": {
      "figma-scaler": {
        "node_id": "1:23",
        "usage_count": 47,
        "rgba": { "r": 37, "g": 99, "b": 235, "a": 1 }
      }
    }
  }
}
```
