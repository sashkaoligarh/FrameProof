# Code of Conduct

## Our Standards

This project is committed to a respectful, inclusive, and technically constructive community. Participants are expected to:

- Be considerate and professional in issues, reviews, discussions, and other project spaces.
- Critique ideas and code rather than people.
- Respect differing experience levels, identities, viewpoints, and communication styles.
- Protect private design data, credentials, and vulnerability details.
- Accept maintainers' reasonable decisions about project scope and moderation.

Harassment, discrimination, threats, sexualized attention, deliberate intimidation, doxxing, credential disclosure, and sustained disruptive behavior are not acceptable.

## Enforcement

Report conduct concerns through GitHub's reporting and moderation tools. Do not publish sensitive personal details in a public issue. Report security matters through GitHub private vulnerability reporting as described in [SECURITY.md](SECURITY.md). If private reporting is unavailable, open a public issue only to request a private channel and include no vulnerability details, secrets, private file IDs, screenshots, or design artifacts.

Maintainers may edit or remove content and may warn, temporarily restrict, or permanently ban participants whose behavior violates this policy. Reports will be handled as privately as the platform and a fair review permit. Retaliation against a reporter is prohibited.

This code applies in project spaces and when someone is publicly representing the project.
